# Ryan Workspace Product Direction

This file defines the global product direction for this AI workspace.

Project-specific PRDs should live inside each project folder and override this file when needed.

---

## Purpose

Build AI-operable business systems that help Ryan manage, market, automate, and scale multiple ventures.

Core projects:

- WEEN: wellness ecosystem and growth-support product system
- CHILLAB: nightlife, cocktail bar, customer experience, event/content system
- HUONG: men’s fashion retail, offline sales optimization, content and commerce workflow

---

## Product Principles

Prioritize:

- practical execution
- clear business value
- trust-building systems
- reusable workflows
- automation readiness
- simple MVP before complex scaling

Avoid:

- hype-first ideas
- bloated dashboards
- vague strategies with no next action
- unnecessary software complexity

---

## AI Workspace Requirements

The workspace should help agents:

- understand project context quickly
- reuse existing files before creating new ones
- keep brand and operational rules consistent
- generate code, content, SOPs, and automation plans from structured context
- reduce repeated prompting across projects

---

## Default Execution Flow

Before implementing a feature or system:

1. Read `MASTER.md`.
2. Read root `AGENTS.md`.
3. Read project-level `AGENTS.md` if available.
4. Read relevant `DESIGN.md`, `PRD.md`, `TASK.md`, or spec files.
5. Make the smallest useful change first.
6. Explain only what changed and what to test next.

---

## Long-Term Goal

Create a reusable AI-native operating system for Ryan’s businesses:

- persistent context
- repeatable workflows
- project-specific instructions
- scalable content systems
- automation-ready documentation
- codebases that Codex/agents can maintain efficiently
