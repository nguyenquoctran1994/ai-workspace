# Ryan Workspace Design System

This file defines global design principles for all projects in this AI workspace.

Project-specific design rules should live inside each project folder and override this file when needed.

---

## Global Design Direction

Default visual direction:

- premium minimalism
- clean spacing
- high readability
- mobile-first layouts
- modern startup feeling
- reusable visual systems

Avoid:

- cluttered layouts
- random gradients
- low contrast text
- inconsistent spacing
- excessive animation
- visual styles that conflict with the project brand

---

## Project Style Separation

Do not mix visual identities across projects.

- WEEN: Japanese biotech wellness, deep navy, white, cyan glow, premium trust.
- CHILLAB: realistic Vietnamese nightlife, deep purple/blue/red ambient lighting, grounded venue realism.
- HUONG: clean men’s fashion retail, practical officewear, authentic brand-dealer positioning.

When working on one project, read its project-level design files before generating UI, images, or layouts.

---

## UI Rules

Use:

- clear hierarchy
- reusable sections
- consistent spacing
- responsive components
- simple interaction patterns
- accessible contrast

Avoid:

- unnecessary dashboards
- overcomplicated user flows
- fake data-heavy screens
- excessive decorative effects

---

## Image / Visual Asset Rules

Before generating visuals:

- identify the target project
- follow the project’s approved style reference
- avoid generic AI fantasy aesthetics
- preserve brand logo proportions when logo is provided
- keep Vietnamese text accurate when used

---

## Agent Behavior

Agents should:

- check existing visual rules before creating new styles
- preserve consistency with approved references
- avoid redesigning logos unless explicitly requested
- prefer reusable design systems over one-off designs
